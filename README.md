A Software as a Service (SaaS) platform that generates creative videos using multiple AI models. The platform creates unique ideas, converts them into images, videos, and adds AI-generated music and voice narration.

## Features

### Core Features
- **User Authentication & Management**: Secure login, registration, and profile management
- **Project Dashboard**: Create, view, and manage all your video projects in one place
- **Subscription Tiers**: Free, Pro, and Enterprise plans with different capabilities
- **API Access**: Programmatic access for developers (Enterprise plan)

### AI-Powered Video Creation
- **Idea Generation**: Creative concepts generated by GPT-4o
- **Image Generation**: High-quality images created by Flux Image Pro
- **Video Animation**: Dynamic videos produced by Kling AI
- **Music Generation**: Custom soundtracks composed by SonAuto
- **Voice Narration**: Professional voiceovers using OpenAI TTS
- **Automatic Composition**: All elements combined seamlessly with FFMPEG

### Technical Features
- **Background Processing**: Asynchronous task handling with Celery
- **Cloud Storage**: Secure storage for all generated media
- **Analytics Dashboard**: Track usage and performance metrics
- **Responsive Design**: Works on desktop, tablet, and mobile devices

### Backend
- **Framework**: Flask 2.3.3
- **ORM**: SQLAlchemy 2.0.23
- **Authentication**: Flask-Login, JWT
- **Task Queue**: Celery 5.3.4 with Redis 5.0.1
- **API**: Flask-RESTful, Flask-RESTX

### Frontend
- **HTML/CSS/JS**: Bootstrap 5.3
- **Templates**: Jinja2
- **Interactive UI**: JavaScript with Fetch API
- **Responsive Design**: Mobile-first approach

### Database
- **Development**: SQLite
- **Production**: PostgreSQL
- **Migrations**: Flask-Migrate (Alembic)

### Infrastructure
- **Containerization**: Docker, Docker Compose
- **Storage**: Local filesystem / AWS S3
- **Caching**: Redis

### AI Services
- **Text Generation**: OpenAI GPT-4o
- **Image Generation**: Flux Image Pro via Replicate
- **Video Generation**: Kling AI via Replicate
- **Music Generation**: SonAuto API
- **Voice Synthesis**: OpenAI TTS

## Installation

### Quick Start with Docker

1. **Clone the repository**:
```bash
git clone https://github.com/All-About-AI-YouTube/ai_video_pipeline.git
cd ai_video_pipeline
```

2. **Create environment file**:
```bash
cp .env.example .env
```

3. **Edit the `.env` file** with your API keys and configuration:
```ini
# Required API Keys
OPENAI_API_KEY=your_openai_key
REPLICATE_API_KEY=your_replicate_key
SONAUTO_API_KEY=your_sonauto_key

# Flask Configuration
FLASK_SECRET_KEY=generate_a_secure_random_key
FLASK_DEBUG=1

# Optional: Stripe Configuration (for payments)
STRIPE_PUBLISHABLE_KEY=your_stripe_publishable_key
STRIPE_SECRET_KEY=your_stripe_secret_key
STRIPE_WEBHOOK_SECRET=your_stripe_webhook_secret

# Optional: AWS Configuration (for S3 storage)
AWS_ACCESS_KEY_ID=your_aws_access_key
AWS_SECRET_ACCESS_KEY=your_aws_secret_key
AWS_S3_BUCKET_NAME=your_s3_bucket_name
AWS_REGION=us-east-1
```

4. **Build and start the Docker containers**:
```bash
docker-compose up -d
```

5. **Access the application** at [http://localhost:5000](http://localhost:5000)

## Development

### Local Development Setup

1. **Create and activate a virtual environment**:
```bash
python -m venv venv
source venv/bin/activate  # On Windows use: venv\Scripts\activate
```

2. **Install required packages**:
```bash
pip install -r requirements-web.txt
```

3. **Set up environment variables**:
```bash
# Linux/macOS
export FLASK_APP=app.py
export FLASK_DEBUG=1

# Windows (Command Prompt)
set FLASK_APP=app.py
set FLASK_DEBUG=1

# Windows (PowerShell)
$env:FLASK_APP = "app.py"
$env:FLASK_DEBUG = "1"
```

4. **Create necessary directories**:
```bash
mkdir -p static/image static/video static/music static/voice static/final prompts
```

5. **Start Redis** (in a separate terminal):
```bash
redis-server
```

6. **Run the Flask application**:
```bash
flask run
```

7. **Start the Celery worker** (in another terminal):
```bash
celery -A tasks.celery worker --loglevel=info
```

## API Documentation

The AI Video Generator provides a RESTful API for programmatic access to the platform. This API is available to users on the Enterprise plan.

### Authentication

All API requests require authentication using an API key. You can obtain your API key from your account settings.

```bash
curl -X GET "https://api.aivideogenerator.com/api/v1/projects" \
  -H "X-API-KEY: your_api_key_here"
```

### API Endpoints

#### Projects

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/v1/projects` | GET | List all projects |
| `/api/v1/projects` | POST | Create a new project |
| `/api/v1/projects/{id}` | GET | Get a project by ID |
| `/api/v1/projects/{id}` | PUT | Update a project |
| `/api/v1/projects/{id}` | DELETE | Delete a project |

#### Generation

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/v1/generation/generate` | POST | Start a video generation task |
| `/api/v1/generation/status/{task_id}` | GET | Get the status of a generation task |

### Example: Creating a Project

```bash
curl -X POST "https://api.aivideogenerator.com/api/v1/projects" \
  -H "X-API-KEY: your_api_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "My New Project",
    "idea": "A serene mountain landscape with flowing rivers and lush forests",
    "prompt": "Photorealistic mountain landscape, flowing rivers, lush green forests, dramatic lighting, 8k resolution"
  }'
```

### Example: Generating a Video

```bash
curl -X POST "https://api.aivideogenerator.com/api/v1/generation/generate" \
  -H "X-API-KEY: your_api_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "project_id": 123,
    "step": "all"
  }'
```

For complete API documentation, visit our [GitHub repository](https://github.com/All-About-AI-YouTube/ai_video_pipeline).

## Troubleshooting

### Common Issues

#### Installation Issues

**Problem**: Docker containers fail to start

**Solution**: Check if Docker and Docker Compose are properly installed and running. Also, ensure that the required ports (5000, 6379) are not being used by other applications.

```bash
# Check Docker status
docker --version
docker-compose --version

# Check if ports are in use
lsof -i :5000
lsof -i :6379
```

**Problem**: Missing API keys

**Solution**: Ensure that you have added all required API keys to your `.env` file. Without these keys, the AI services will not work.

#### Generation Issues

**Problem**: Video generation fails

**Solution**: Check the logs for error messages. Common issues include:
- Invalid or expired API keys
- Network connectivity problems
- Insufficient resources (memory, disk space)

```bash
# Check Celery worker logs
docker-compose logs worker

# Check application logs
docker-compose logs web
```

**Problem**: Slow generation times

**Solution**: Video generation is resource-intensive. Consider:
- Upgrading your hardware
- Optimizing your prompts
- Reducing video resolution or duration

#### Web Interface Issues

**Problem**: Can't access the web interface

**Solution**: Ensure that the Flask application is running and accessible:

```bash
# Check if the web container is running
docker-compose ps

# Try accessing with curl
curl http://localhost:5000
```
